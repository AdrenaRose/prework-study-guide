HTML

  • The head element contains information about the webpage. This information is not visible to the user and is called metadata.
  • Metadata tags include:
    • <"title">
    • <"style">
    • <"base">
    • <"link">
  • The body element contains the content that we want the user to see.
  • There may be up to 6 heading <"h1,h2,h3,etc"> elements in the body of one HTML document.
  • A child element is one that is nested within other elements. Those other elements are called parent elements.
  • Semantic HTML elements are those that use the name of the element to describe its contents.

CSS

  • CSS = Cascading Style Sheets
  • Defines colors, aesthestics, visual layout, and organization of a webpage
  • Inline CSS is used when a developer wants to style an element directly in the HTML file
    • Useful for quick and permanent changes
    • Applied to a specific HTML tag, using a style attribute with a CSS rule
    • Each inline style must be edited individually whenever there is a design change
    • Example: [h1 style="color:red;"] Prework Study Guide[/h1]
  • Internal CSS style sheet
    • CSS style sheet embedded directly in the HTML file
    • Use a [style] element in the [head] element
    • Rules apply to that page only
    • Classes and ids are used to identify elements and style multiple elements in the page code
  • External CSS style sheet
    • Most common way to style a webpage
    • Preferred because it allows developers to keep all their CSS rules in a separate file, making design changes easier
    • To use, the external style sheet is linked in the HTML's head element using a link element
  • Breakdown of a CSS rule: div (selector) {color: (property) blue (value); (declaration) text-align: (property) justify (value); (declaration)}
    • Selector defines the element or attributes to which the declarations will apply
    • Declarations contain the CSS property we want to apply and the value of that property
    • Two or more declarations in a row is called a declaration block
  • The margin property dictates how much space we want around the *outside* of an element.
  • The padding property adds space around the content *inside* an element. If padding is increased, the size of the entire element is increased.
  • Consider every HTML element like a box
    • CSS box model - each box (element) consists of margins outside the element, and the element's borders, padding, and content.
    • Content, padding, and border are *inside* the element.
  • The * (wild card symbol) can be used as a selector to apply a rule to ALL elements on a page.
  • Two selectors can be combined into a single CSS rule with a comma
  • Class attributes are used when we would like to apply styling to certain elements, but not all elements of the same type
  • Class attributes are then used as CSS selectors so that we can assign properties to them

Javascript

  • Javascript is a high-level dynamic programming language that allows users to interact with websites
  • Javascropt file must be linked to html file, similarly to CSS, but using the script element at the bottom of the body
    • Link is placed on the line directly above the closing body element - this is to ensure that the html is able to load on the page before the browser attempts to run the script
  • Document Object Model (DOM) API has the capability to manipulate documents
  • Variables: Containers that store values, declared by the word let followed by the intended variable's name
    • Variable names usually start with a letter, underscore, or dollar sign
    • Numbers can be used after the starting character (Ex. temp99)
    • JS is case sensitive, so A-Z and a-z can also be used and will be considered unique
  • Semicolons are the end of a line indicates where the statement ends
  • Naming and declaring variables are not the same thing and can be accomplished on one line by typing: let (declaration) myVariable (variable name) = "value of variable" (assigns value to variable)
  • Variables are retrieved by calling the variable name
  • Variable names can be changed later in the code by typing: variableName = "New value"
  • Data Types
    • String: a sequence of text signified by the use of single or double quote marks
    • Number: A number (obviously). Numbers do not have quotes around them
    • Boolean: true/false value. True and false are special values and do not need quotes
    • Array: a structure that allows multiple values to be stored in a single reference
      • Example: let myVariable = [1, 'Bob', 'Steve', 10]
      • Members of arrays are referred to by number, starting with 0. Example: myVariable[0] would return the value 1
    • Object: This can be anything. Everything in JS is an object and can be stored in a variable
  • Operators: mathematical symbols that produce a result based on two values (or variables)
    • Addition (+): add two numbers or combine two strings
    • Subtraction, Multiplication, Division (-, *, /): Do basic math
    • Assignment (=): Assigns a value to a variable
    • Strict Equality (===): Performs a test to see if two values are equal and returns a boolean result
    • Not, Does-not-equal (!, !==): (!) Returns the logically opposite value of what it precedes, like turning true into false. (!==) Tests whether two values are not equal
  • Conditionals: code structure that tests if an expression returns true or not, commonly if...else
    • if(variableName === "value") { alert ("text"); } else { alert ("text"); }
    • The expression inside the if() is the test and uses the equality operator to compare the variableName with the variableValue to see if they are equal
  • Functions: a way of packaging functionality that you wish to reuse
    • Can be built in, or created by developer using function-fuctionName(argument(s))
    • Return statement tells the browser to return the result variable out of the function so it is available to use. Variables defined inside functions are ONLY available inside those functions (variable scoping)
  • Events
    • Event handlers are code structures that listen for activity in the browser, and run code in response
    • Example: click event - this event is fired by the browser when you click on something with your mouse
      • document.querySelector("button").addEventListener("click", function() { alert("Ouch! Stop poking me!"); });
      • In this statement, the button element is selected. addEventListener function adds an event to listen for, in this case, click. Then, a function to run when the event happens is added. In this case, the alert
      • If a function that is passed to addEventListener() has no name, it is an anonymous function. Can also be written as:
        • document.querySelector("button").addEventListener("click", () => { alert("Ouch! Stop poking me!"); });
  • Web Storage API
    • locaStorage.setItem(value, variable); - this saves the data in the browser to be retrieved later
  • Null is a special value in JS that refers to the absence of a value
